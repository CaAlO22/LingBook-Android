## 执行步骤

### 步骤 1: 提交更新检查功能
```bash
git add app/src/main/AndroidManifest.xml \
        app/src/main/java/com/lingji/app/MainActivity.kt \
        app/src/main/res/values/strings.xml \
        app/src/main/java/com/lingji/app/data/remote/UpdateChecker.kt \
        app/src/main/java/com/lingji/app/ui/components/UpdateDialog.kt \
        app/src/main/java/com/lingji/app/ui/viewmodel/UpdateViewModel.kt

git commit -m "feat: add in-app update checker via GitHub Releases"
```

### 步骤 2: Bump 版本号到 1.2.0
编辑 `app/build.gradle.kts`:
- `versionCode = 4` → `versionCode = 5`
- `versionName = "1.1.2"` → `versionName = "1.2.0"`

```bash
git add app/build.gradle.kts
git commit -m "chore(release): bump version to 1.2.0 (versionCode 5)"
```

### 步骤 3: 安装 gh CLI
```bash
winget install --id GitHub.cli -e --accept-source-agreements --accept-package-agreements
```
安装后需重启 shell 或刷新 PATH。

### 步骤 4: 认证 gh CLI
```bash
gh auth login
```
选择 GitHub.com → HTTPS → 浏览器认证。

### 步骤 5: 推送代码和 tag
```bash
git push origin main
git tag v1.2.0
git push origin v1.2.0
```

### 步骤 6: 构建 Release APK
```bash
./gradlew :app:assembleRelease
```

### 步骤 7: 创建 GitHub Release
```bash
gh release create v1.2.0 \
  --repo CaAlO22/LingBook-Android \
  --title "v1.2.0" \
  --notes "## v1.2.0

### 新功能
- 应用内更新检查：启动时自动检查 GitHub Releases 获取新版本
- 支持一键下载并安装 APK 更新

### 改进
- 新增安装包权限以支持应用内更新" \
  app/build/outputs/apk/release/app-release.apk
```
